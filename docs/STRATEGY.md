### "국내 상장리츠 투자자의 의사결정 레이어"

> 살아있는 전략 문서. 방향이 바뀌면 이 파일을 갱신한다. 최종 갱신: 2026-07.

## I. 냉정한 현 위치 진단

전략은 정확한 자기인식에서 시작한다. 지금 리츠온이 실제로 가진 것과 비어 있는 것을 구분한다.

### 이미 완성된 자산 (해자의 골격)

| 자산 | 전략적 의미 |
|---|---|
| **검증된 팩트 레이어** — 실배당 TTM·P/NAV·품질배지, 25종목 전부 출처+기준일 | 네이버금융·증권사앱·KAREIT·유튜브 어디에도 없는 유일 지표. "왜 여기냐"의 10초 답이 숫자 3개로 존재 |
| **자동 파이프라인** — fetch→validate→build 일 2회, SSG로 페이지 한계비용 ≈ 0 | 1인 운영에서 콘텐츠를 무한 증식시킬 수 있는 엔진 |
| **축적형 데이터** — `price-history.json`(1년치 일별, 매일 성장), `filings.json`, `changes.json`, `snapshot.json` | 시간이 지날수록 가치가 커지는 유일한 자산군. '추세·변동성·diff' 기능의 원재료가 매일 쌓이는 중 |
| **provenance 문화** — /about 방법론, "산정중"으로 비우는 정직성, 확정 아님 명시 | LLM 검색 시대의 결정적 해자. AI 오버뷰·퍼플렉시티는 출처·기준일 명확한 정적 표를 인용함 |
| **정체성 규율** — 추천·순위·점수 없음, pfAutoRecommend도 "후보로만"으로 완화 완료 | 규제 방어인 동시에 브랜드 신뢰. 이전 전략의 #1 법적 리스크가 이미 제거됨 |
| **분산 배선** — 섹터 페이지 8종, OG 카드, .ics, 404, 변화 로그 | 검색 유입·공유 바이럴·재방문의 기초 배관 완성 |

### 아직 비어 있는 3대 구조적 공백

1. **"과거"는 있는데 "지속가능성"이 없다.** 사이트는 *받았던* 배당(TTM)을 보여주지만, P1 페르소나(준은퇴 현금흐름 설계자)의 진짜 질문 "이 배당이 깎일 위험은 없나?"엔 답하지 못한다. 답할 재료(배당성향>100% 플래그, LTV, WALE, 고정금리비중, 12개월 내 만기)는 이미 `reportDetail`에 다 있다. **합성만 안 됐을 뿐이다.**

2. **캘린더가 "예상"에 머물러 있다.** `.ics`는 결산월 말일 가정. 습관 루프의 진짜 앵커는 DART에서 확정된 배당기준일·지급일이다. 이걸 파싱하는 순간 리츠온은 "가끔 보는 곳"에서 "배당 시즌마다 반드시 여는 곳"이 된다.

3. **리텐션 루프가 열려 있지 않다.** 관심리츠 토글은 있지만 (a)기기 간 동기화 없음 (b)알림이 실제로 발사되지 않음 (c)실데이터 기반 포트폴리오 추적 없음. 방문자를 *사용자*로 전환하는 마지막 고리가 비어 있다.

> **한 줄 진단:** 리츠온은 "리츠를 공부하는 곳"에서 "리츠 팩트를 확인하는 곳"까지 왔다. 다음 목적지는 **"리츠 결정을 내리고 그 결정을 추적하는 곳"**이다.

## II. 궁극적 방향성

### 비전 한 문장

> **한국 상장리츠에 대해, '이 배당이 진짜인가 · 이 가격이 싼가 · 언제 무엇이 바뀌었나'를 검증 가능한 형태로 답하는 유일한 의사결정 레이어.**

### 3-레이어 성숙 모델

```
   [ HABIT LAYER ]   ← 최종 목적지 (2027~)
   배당 시즌마다 여는 곳. 알림·확정일정·내 포트폴리오 추적.
   "내가 가진 리츠에 무슨 일이 생기면 리츠온이 알려준다"
        ▲
   [ CONTEXT LAYER ]  ← 다음 1년의 주전장 (2026 H2~2027 H1)
   팩트에 '맥락'을 더한다. 배당 지속가능성, 시계열 추세,
   diff 저널리즘, 비교축(vs 맥쿼리·ETF·예금)
        ▲
   [ FACT LAYER ]    ← 완성됨 (2026 H1)
   TTM·P/NAV·리스크 팩트 + provenance
```

**핵심 판단: Context Layer가 승부처다.** Fact Layer는 방어 가능하지만 "한 번 보면 되는" 성격이라 재방문을 못 만든다. Context Layer(특히 diff·지속가능성)가 재방문 이유와 SEO 롱테일을 동시에 만들고, 그게 쌓여야 Habit Layer가 의미를 가진다.

### 절대 넘지 않을 선 (정체성 = 규제 방어 = 브랜드)

- ❌ 매수·매도·목표주가·종합점수·순위 — 영원히 안 함
- ❌ 출처 없는 수치 — "산정중"으로 비움
- ✅ 사실 서술 + 기준일 + 원문 링크 — 이게 제품
- ✅ 모든 평가성 언어("안정/위험")는 규칙과 근거를 함께 노출

## III. 전략 기둥 5개

### 기둥 1 — DEPTH: 팩트에서 '맥락'으로
- **왜.** 유일한 미답 질문 "이 배당 지속되나?"에 답. 재료는 이미 있음.
- **무엇.** 배당 지속가능성 신호(점수 아닌 체크리스트), 경상배당 추세, P/NAV 맥락화(왜 싼지).
- **어떻게.** `lib/sustainability.mjs` 빌드타임 계산. 새 데이터 불필요. 코드만. "종합점수 아님 · 사실 서술" 프레이밍 필수.

### 기둥 2 — HABIT: 일회성 방문 → 리듬
- **왜.** 25종목 니치 천장(월 3~10만 PV 추정)은 빈도로만 뚫린다.
- **무엇.** 확정 배당 일정(최우선), 웹푸시 알림, 주간 다이제스트, 내 포트폴리오 추적.
- **어떻게.** Worker(KV/Cron)+Actions로 서버리스. 포트폴리오는 localStorage 우선. 알림·이메일은 사용자 액션 1회 필요.

### 기둥 3 — DISTRIBUTION: 유입 엔진
- **왜.** 헤드쿼리는 네이버금융 독점 → 포기. 지표형·롱테일·시의성·LLM인용 장악.
- **무엇.** 프로그래매틱 페이지 확장, diff 저널리즘, LLM 최적화, 위성 채널(네이버 서치어드바이저·블로그).
- **어떻게.** 대부분 `build-pages.mjs` 확장 = 코드만. 네이버 등록·블로그는 사용자 액션.

### 기둥 4 — TRUST: provenance를 제품이자 해자로
- **왜.** 금융 YMYL에서 신뢰는 전환·재방문·SEO·LLM인용·규제방어를 동시에 좌우.
- **무엇/어떻게.** 3종 세트(어디서·언제·어떻게) 유지. 산정 로직 공개, 이상치 자동 플래그, 교차검증 실패 시 미표시.

### 기둥 5 — SUSTAINABILITY: 규율 있는 수익화
- **왜.** 저품질 수익이 정체성을 파괴하면 해자가 무너짐.
- **순서.** (1)커스텀 도메인 = 애드센스 하드 게이트 (2)애드센스+증권사 CPA 제휴 (3)절대 안 팔 것: 종목 스폰서·리딩방·과장 CPA.
- **어떻게.** 전부 사용자 액션. 개발 몫 거의 없음.

## V. 의사결정 프레임워크

1. **코드만으로 되는가?** → 그렇다면 Phase A에서 즉시.
2. **재방문을 만드는가?** → 재방문(diff·알림·일정) 우선. 니치에선 빈도가 왕.
3. **정체성을 지키는가?** → 추천·점수로 미끄러지면 폐기. "사실 서술 + 근거 노출"로 재설계.
4. **provenance가 있는가?** → 출처 못 대는 수치는 안 만듦.
5. **1인 운영 원가를 지키는가?** → 서버 상태가 꼭 필요한가로 판단. 대부분 정적+Worker로 커버.
