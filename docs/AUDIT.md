# AUDIT.md — 리츠온 제품·데이터 감사 (2026-06-19)

대상: https://seonghoonkim2.github.io/reits-on/ · 정적(GitHub Pages) + Cloudflare Worker API(D1/KV/cron) 하이브리드.

## 1. 확인된 현황(사실)
- 단일 `index.html`(임베드 seed-data) + Worker API(`/v1/market|reits|filings`, 일배치 cron, DART 공시 자동수집 가동).
- localStorage(`reiton_*`): 관심/비교/포트폴리오/체크리스트/설정. `#~t=` 딥링크.
- 시세(가격) **미연동**: 사용자 입력 기반 예상 수익률.
- SSG 종목 페이지 `/r/<ticker>/` 25개 + sitemap + cron, 웹푸시(gated), 공시 키워드 해설, PWA(manifest+SW), 바텀시트 focus trap/ESC.

## 2. 발견한 문제 → 조치
| 문제(증거) | 사용자 영향 | 조치 |
|---|---|---|
| “자동 추천”/“매달 배당받는” | 투자권유 오인·입금시점 오해 | **수정**: “조건 기반 조합 예시”·“배당기준월 분산”+지급 2~3개월 시차 명시 |
| “배당 많은 순” 정렬 | 기준 불명확 | **수정**: “연환산 DPS 순(추정)” |
| “현재가 기준 수익률”(가격 미연동) | 실시간 오인 | **수정**: “입력가 기준 예상 수익률” |
| 섹터 “안정 편/수요 안정/분산 효과” | 검증불가 안전성 평가 | **수정**: 성격 라벨(장기임대중심 등) + “정량등급·안전성평가 아님” |
| AUM/WALE/LTV/임차인/차입만기 **데이터 부재** | 핵심 팩트 제공 불가 | **데이터 모델 신설**: facts(provenance), 미확보는 “자료 확인 필요” |
| 수치에 기준일·출처·상태 미분리 | 다른 시점 혼용 위험 | **수정**: provenance 객체(value·asOf·status·sourceUrl) + 상세 ‘핵심 팩트’ |
| 데이터가 임베드에만 존재(검증 없음) | 오류 데이터 위험 | **수정**: data/*.json + schema + validate + CI 게이트 |

## 3. 데이터 오해 가능성(잔존, 라벨로 완화)
- 자산총계(assetBn) ≠ 연결 AUM: facts.aum note에 명시.
- 배당기준월 ≠ 지급월: 인컴 탭·캘린더·상세에 반복 고지(payMonths는 unavailable).
- 평균 배당수익률: 협회 집계(단순평균 아님)·특별배당 포함 여부 종목별 상이 — 주석 표시.
- 연환산 DPS: 최근 1회×횟수 단순 추정(특별배당 미구분) — “추정” 라벨.

## 4. 규제 검토 필요(단정 금지)
- 조건 기반 시나리오·정렬·향후 LLM 요약의 투자권유/유사투자자문 해당 여부.
- 광고·제휴 고지(반영됨)와 시나리오 결과의 분리(반영됨).
- 향후 클라우드 동기화 시 개인정보 고지 모드 분리 + 동의.
