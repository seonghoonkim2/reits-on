# LLM_RFC.md — 공시·IR 요약 파이프라인 (설계, 승인 전 PoC 한정)

상태: **설계만**. 외부 LLM API 호출은 “중대 변경”으로 별도 승인 필요. 현재 운영은 키워드 기반 `filingExplain()`(날조 0)로 대체 중.

## 1. 목표·비목표
- 목표: 공시 원문에서 **사실만** 구조화 추출 → 근거(evidenceId) 연결된 요약을 정적 JSON으로 배포.
- 비목표: 매수·매도·보유 의견, 목표주가, “유망/저평가/안전” 같은 평가, 인과 단정.

## 2. 파이프라인 (브라우저에서 LLM 직접 호출 금지)
```
공시 목록(DART list.json) → 원문 HTML/PDF 저장 → 문서유형 분류
→ 텍스트·표 추출 → section 기반 chunk → 구조화 사실 추출(LLM)
→ 숫자 검증(결정론적) → 금칙어 검사 → 정적 JSON 배포
```
LLM 호출은 GitHub Actions(서버 측, 시크릿 키)에서만. 결과는 검증 통과분만 커밋.

## 3. 문서·청크 식별자(provenance 보존)
각 청크: `{rcept_no, report_nm, filed_at, filer, url, page, sectionTitle, paraId, hash, prevId, nextId, sourceType: "filing"|"ir"}`
- 임의 토큰 길이로 자르지 않음. 제목/문단/표/주석 단위.
- 표와 그 주석을 분리하지 않음. 숫자의 단위·기준일을 같은 청크에 보존.

## 4. 숫자 환각 방지(결정론 게이트)
- 모든 numericClaim에 evidenceId 필수. 원문에 없는 숫자 생성 시 **배포 차단**.
- 원문 숫자 ↔ 추출 숫자 정규화 비교(억/백만/원 혼용 금지, 단위변환은 별도 순수함수).
- 전기대비 증감률은 **코드에서** 계산(LLM이 계산하지 않음).
- 날짜·보고기간 자동 검증. 실패 시 “요약 생성 실패 · 원문 확인 필요”.

## 5. 구조화 출력 스키마
```json
{
  "summaryType": "filing_key_items",
  "documentId": "20260319000123",
  "facts": [
    { "category": "refinancing", "statement": "…", "evidenceIds": ["p12-s3-table2"], "numericClaims": [
      { "label": "차입금리", "value": 4.2, "unit": "%", "asOf": "2026-03-19", "evidenceId": "p12-s3-table2" } ] }
  ],
  "dividendDrivers": [ { "factor": "…", "direction": "영향 가능", "evidenceIds": ["p4-s1"] } ],
  "itemsToVerify": ["…"],
  "prohibitedExpressionDetected": false
}
```

## 6. 프롬프트(요지)
**system**: “너는 한국 상장리츠 공시 추출기다. 제공된 근거 청크만 사용한다. 근거 없으면 ‘확인 불가’. 매수·매도·보유 의견, 목표주가, 적정가치, ‘긍정적/부정적/유망/저평가/안전’ 표현 금지. 공시 사실과 해석을 분리. 반복 임대수익과 일회성 매각이익을 구분. 변경 전/후 수치가 있으면 둘 다. 배당 영향은 인과 단정 없이 ‘영향 가능 요인’으로. 문장마다 evidenceId 연결. 주어진 JSON Schema를 벗어나지 말 것.”
**user**: `{문서 메타 + chunks[] (각 {paraId, text})}` → “위 근거만으로 schema에 맞춰 추출.”

## 7. UI 문구(평가 표현 배제)
- “AI 공시 요약” → **“공시 핵심 항목 자동 정리”**
- “리스크 분석” → **“추가 확인이 필요한 공시 항목”**
- “배당 전망” → **“공시에 나타난 배당 변동 요인(영향 가능)”**
- 결과 인근: “자동 생성 요약입니다. 누락·해석 오류 가능. 투자 판단 전 공시 원문을 확인하세요.” + 자동생성 배지.

## 8. 규제 검토 포인트
- 자동 요약이 유사투자자문/투자권유로 오인될 소지 → 평가·전망 표현 배제, 사실+근거+원문 CTA 우선.
- 모델/비용/실패율, 개인정보(공시 원문엔 개인정보 거의 없음), 캐시·재배포 권리 검토.

## 9. PoC 범위(승인 시)
1종목·5건 공시로 추출→검증→스키마 출력까지. 정확도(숫자 일치율)·금칙어 0 확인 후 확장.
