# SCENARIO_ENGINE_RFC.md — 조건 기반 배당 조합 시나리오

상태: MVP는 **설명 가능한 휴리스틱**(현재 가동: 그리디 배당월 커버). “자동 추천/최적”이라 부르지 않음 — “조건 기반 조합 예시”.

## 1. 표현·규제 원칙
- “추천/유망/최적/저평가/안전/매수” 금지. 결과는 **시나리오 예시**.
- 기본값으로 특정 종목을 밀지 않음. 입력 조건에 따라 **기계적으로** 계산.
- 사용한 입력·데이터 기준일·제약 충족 여부·월별 부족액·집중도를 모두 표시.
- 증권사 제휴 CTA를 결과 바로 옆에 배치하지 않음.

## 2. 입력(사용자가 직접)
총 투자금, 사용 가격(입력가), 예상 DPS 방식, 세전/세후, 목표 월 현금흐름, 목표 수익률, 종목·섹터·임차인·해외 최대 비중, 최소/최대 종목 수, 특별배당 포함 여부, 배당기준월/지급월 기준.

## 3. MVP 휴리스틱(설명 가능)
1. 입력 조건으로 후보 필터(데이터 없는 종목은 해당 제약에서 제외 + 경고).
2. 비어 있는 배당기준월 우선 채움(그리디 set-cover).
3. 종목·섹터·임차인 집중도 상한 적용.
4. 월별 목표 대비 부족액을 줄이는 종목을 순차 배정.
5. 반복 교환(local search)으로 월별 편차 감소.
- 현재 코드: `pfAutoRecommend()`가 2번(그리디 커버)+균등분배. 3~5는 P1 확장.

## 4. 고급안(MILP, P2)
변수: `x_i`(종목 수량, 정수≥0), `y_i`(포함 여부, 0/1), `d_m`(월별 목표 대비 절대편차).
목적: `min Σ_m d_m + λ1·종목집중 + λ2·섹터집중 + λ3·임차인집중`.
제약: 총매입 ≤ 예산, 기대수익률 ≥ 목표, 종목/섹터/임차인 비중 ≤ 상한, 종목 수 [min,max], `x_i` 정수.
- **선형 절대편차** 우선(제곱편차보다 단순·설명 가능). `d_m ≥ target_m − Σ inflow; d_m ≥ Σ inflow − target_m`.

## 5. 25종목 규모 도구 비교
| 방식 | 번들 | 재현성 | 설명가능성 | 결론 |
|---|---|---|---|---|
| 투명 휴리스틱 | 0 | 높음 | 매우 높음 | **MVP 채택** |
| glpk.js(클라 MILP) | ~수백 KB(wasm) | 높음 | 중간(블랙박스) | P2, lazy import |
| 서버/오프라인 Python | 0(클라) | 높음 | 높음 | 대량/정밀 시 |
→ 25종목에선 휴리스틱이 비용·설명가능성 모두 우위. MILP는 “정밀 시나리오” 옵션으로만.

## 6. 결과 UI
- “최적 포트폴리오”라 부르지 않음. **2~3개 시나리오** 제시: ①월 현금흐름 평탄화, ②집중도 제한, ③수익률 가정 중심.
- 표시: 입력·기준일, 제약 충족 여부, 월별 부족액, 종목/섹터/임차인 집중도, 특별배당 포함 여부, “가정이 바뀌면 결과가 달라짐”.

## 7. 규제 검토
조건 입력→기계적 산출→시나리오 표현이 투자권유에 해당하는지(법률 검토 필요). 완화: 다중 시나리오·제약 미충족/데이터 누락 표시·원문 확인 CTA·매수 CTA 배제.
